# CONTRIBUTING

Спасибо за интерес к проекту.

## Локальный запуск для разработки

1. Установите зависимости из `requirements.txt`.
2. Для проверок качества установите `requirements-dev.txt`.
2. Создайте `.env` на основе `.env.example`.
3. Запустите `python bot.py`.

## Рекомендации по изменениям

- Старайтесь держать изменения модульными (не смешивать рефакторинг и фичу в одном PR).
- Для новых настроек обновляйте:
  - `config.py`
  - `.env.example`
  - `docs/CONFIG.md`
- Для новых команд обновляйте `docs/COMMANDS.md` и `README.md`.
- Для изменений архитектуры обновляйте `docs/ARCHITECTURE.md`.

## Минимальный чеклист перед PR

- Код запускается локально без ошибок импорта.
- Базовый сценарий `/generate` работает.
- Если трогали `/download`, проверен хотя бы один успешный download flow.
- Документация обновлена.

## Инструменты качества

Проект использует единые правила для форматирования и проверок:

- `python -m ruff check .`
- `python -m ruff format --check .`
- `python -m mypy`
- `python -m pytest -q`

Рекомендуется включить pre-commit:

```bash
pre-commit install
```

## Стиль и совместимость

- Проект использует Python 3.10+.
- Не добавляйте в репозиторий секреты (`.env`, токены).
- Не коммитьте runtime-данные пользователей (`presets/*.json`).
