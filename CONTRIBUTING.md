# Участие в разработке

Спасибо за интерес к проекту.

## Локальный запуск для разработки

1. Установите зависимости из `requirements.txt`.
2. Для проверок качества установите зависимости из `requirements-dev.txt`.
3. Создайте `.env` на основе `.env.example`.
4. Запустите `python bot.py`.

## Рекомендации по изменениям

- Держите изменения модульными: не смешивайте крупный рефакторинг и новую функциональность в одном PR.
- Для новых настроек синхронно обновляйте:
  - `config.py`
  - `.env.example`
  - `docs/CONFIG.md`
- Для новых команд обновляйте `docs/COMMANDS.md` и `README.md`.
- Для архитектурных изменений обновляйте `docs/ARCHITECTURE.md`.

## Минимальный чеклист перед PR

- Код запускается локально без ошибок импорта.
- Базовый сценарий `/generate` работает.
- Если изменения затрагивают `/download`, проверяется как минимум один успешный download-flow.
- Документация синхронизируется с изменениями.

## Инструменты качества

Проект использует единый набор проверок:

- `python -m ruff check .`
- `python -m ruff format --check .`
- `python -m mypy`
- `python -m pytest -q`

Рекомендуется включить pre-commit:

```bash
pre-commit install
```

## Стиль и совместимость

- Целевая версия Python: `3.10+`.
- Секреты не добавляются в репозиторий (`.env`, токены, ключи).
- Runtime-данные пользователей не коммитятся (`presets/*.json`, `sessions/runtime.json`).
