# Журнал изменений

## Назначение

Этот файл фиксирует актуальные направления изменений проекта и текущее состояние ключевых подсистем.

## Архитектура

- Проект использует целевую слоистую структуру: `domain/`, `application/`, `infrastructure/`, `presentation/`.
- Handler-слой остается точкой пользовательской оркестрации и работает через выделенные сервисы/use-case модули.
- Архитектурные границы контролируются автоматическими тестами (`tests/test_architecture_boundaries.py`).

## Генерация и prompt editor

- Редактор генерации поддерживает сценарии параметров, LoRA, референсов, Smart Prompt и улучшения результата.
- Отправка и улучшение изображений работают как отдельные flow с собственными runtime-артефактами.
- Поведение callback-путей и UI-контрактов покрывается тестами регрессии handler-слоя.

## Скачивание моделей

- Система поиска поддерживает CivitAI и HuggingFace, фильтры, профили и постраничную навигацию.
- Локальный индекс моделей (`.comfybot_model_index.json`) синхронизируется с результатами загрузок.
- Логика фильтров и совместимости базовых моделей централизуется в профильных policy/utility модулях.

## Runtime и сессии

- Runtime-снимок хранится в `sessions/runtime.json` и восстанавливает пользовательские сессии между перезапусками.
- Типизированная snapshot-модель включает миграцию схемы и совместимость со стабильными ключами состояния.
- Контракт runtime/session контролируется отдельным набором тестов.

## Локализация

- Интерфейс использует каталог локалей на основе `metadata + messages` файлов.
- Текущий набор локалей включает `ru` и `en`.
- Паритет ключей между локалями проверяется автоматическими тестами ресурсов.

## Качество и процесс

- Для кода и документации действует единый quality gate: `ruff`, `mypy`, `pytest` и дополнительные контрактные тесты.
- Репозиторий использует pre-commit и CI-проверки для стабильности ветки.
- Процессные ограничения рефакторинга и ручные проверки описаны в `docs/REFACTOR_GUARDRAILS.md`.
