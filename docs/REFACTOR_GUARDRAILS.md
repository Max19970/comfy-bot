# Ограничения рефакторинга

## Назначение

Документ фиксирует обязательные правила безопасного рефакторинга.
Каждый этап сохраняет работоспособность бота и предсказуемость пользовательских сценариев.

## Протокол завершения этапа

Этап закрывается только после выполнения всех условий:

1. Пройден автоматический quality gate.
2. Подготовлен этапный отчет: что реализовано, какие риски закрыты, что остается в рамках плана.
3. Передан ручной чеклист проверки в Telegram-сценариях.
4. Получено явное подтверждение успешности этапа от ревьюера.
5. После подтверждения создается один целостный commit этапа.

## Quality Gate (обязательный минимум)

```bash
python -m ruff check .
python -m mypy
python -m pytest -q
python -m compileall -q .
```

## Архитектурные границы для новой разработки

Legacy-модули (`core/*`, `handlers/*`, `application/model_downloader.py`, `infrastructure/comfyui_client.py`, `application/smart_prompt_service.py`) остаются зоной поэтапной миграции.

Новый код размещается в целевых слоях:

- `domain/` — предметные сущности и value objects.
- `application/` — use-cases и прикладная оркестрация.
- `infrastructure/` — внешние API, файловая система, адаптеры.
- `presentation/` — UI/transport-адаптеры.

Правило зависимостей:

- `domain` не зависит от `application`, `infrastructure`, `presentation`, `handlers`, `core`.
- `application` не зависит от `presentation`, `handlers`.
- `infrastructure` не зависит от `presentation`, `handlers`.
- `core` не импортирует `handlers`.

Эти ограничения проверяются тестом `tests/test_architecture_boundaries.py`.

## Контракты обратной совместимости

До отдельного миграционного этапа не меняются:

- callback payload-форматы в Telegram-flow;
- ключи runtime/session, используемые для восстановления состояния;
- публичные команды (`/generate`, `/download`, `/models`, `/queue`, `/settings`, `/cancel`);
- формат сохраненных пресетов и `sessions/runtime.json`.

Если контракт требует изменения, задача выделяется в отдельный этап с мигратором и явным ручным сценарием проверки.

## Ручной smoke-чек (минимум на этап)

- `/start` и переходы по главному меню.
- `/generate` -> открытие редактора -> возврат в меню.
- `/download` -> вход в flow -> возврат/отмена.
- `/models` и `/queue` открываются без ошибок.

Расширенный чеклист: `docs/UI_MIGRATION_PLAN.md` (раздел `Матрица ручной проверки`).
