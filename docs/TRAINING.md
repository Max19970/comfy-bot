# Текстовое обучение для новичка

Этот файл совпадает по смыслу с командой `/training` в боте, но даёт больше контекста.

## 1) Что нужно понимать перед стартом

- Бот сам изображения не рисует: он отправляет задачу в ComfyUI.
- Если ComfyUI недоступен, генерация не начнётся.
- Для первой удачной генерации достаточно: checkpoint + Positive prompt.

## 2) Первый запуск без лишней теории

1. Выполните `/models`.
2. Проверьте, что в ответе есть хотя бы один `checkpoint`.
3. Выполните `/generate`.
4. Нажмите `Checkpoint` и выберите модель.
5. Нажмите `Positive` и напишите, что хотите получить.
6. (Опционально) Нажмите `Negative` и укажите, чего избегать.
7. Нажмите `Генерировать`.

Если вы видите превью и кнопку PNG, базовая цепочка работает.

## 3) Минимальная теория параметров

- `Steps`: больше шагов = обычно чище картинка, но дольше.
- `CFG`: насколько строго следовать prompt.
- `Seed`: фиксированное число делает результат более повторяемым.
- `Size`: больше размер = больше нагрузка на VRAM и время.

## 4) Что делать дальше

- Используйте `/repeat`, чтобы быстро получить вариации последней сцены.
- Сохраняйте удачные комбинации через `/presets`.
- Добавляйте стиль через LoRA.
- Используйте Smart Prompt (TIPO), если удобнее писать обычным языком.

## 5) Расшифровка частых сообщений

- `Нет предыдущей генерации. Используйте /generate.`
  - Это нормально для первого запуска.

- `На сервере ComfyUI нет ни одного checkpoint.`
  - Скопируйте checkpoint в папку моделей ComfyUI и снова вызовите `/models`.

- `Ошибка подключения`
  - Проверьте `COMFYUI_URL` и что сервер ComfyUI запущен.

- `Я не ожидаю текст. Используйте /generate, /help или /training.`
  - Бот не находится в шаге ввода текста. Вернитесь к команде/кнопке.
