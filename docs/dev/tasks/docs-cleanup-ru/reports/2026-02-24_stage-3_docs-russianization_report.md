## Что реализовано

- Устранены фактические расхождения, зафиксированные на этапе 2:
  - `docs/SETUP.md`: переменные `COMFYUI_URL` и `ALLOWED_USERS` описаны как опциональные;
  - `docs/CONFIG.md`: добавлен параметр `SMART_PROMPT_FORMAT`;
  - `docs/ARCHITECTURE.md`: раздел runtime приведен к фактическому поведению восстановления;
  - `docs/CHANGELOG.md`: документ переработан в формат актуального состояния.
- Проведена языковая унификация документации: `CONTRIBUTING.md`, `SECURITY.md`, `docs/TROUBLESHOOTING.md`, `docs/REFACTOR_GUARDRAILS.md` приведены к русскому языку и настоящему времени.
- По запросу пользователя локализация шаблонов `docs/dev/tasks/_templates/` отменена, шаблоны возвращены к исходному виду.
- Исправлены редакторские несоответствия в `docs/COMMANDS.md` (дубли нумерации шагов).
- Обновлены реестр задачи (`inventory`), план и отчет этапа 3 с фиксацией пользовательской корректировки.

## Шаги ручной проверки

1. Откройте `docs/SETUP.md`, `docs/CONFIG.md` и `docs/ARCHITECTURE.md`, затем проверьте, что фактические расхождения этапа 2 закрыты.
2. Откройте `docs/dev/tasks/_templates/task_plan_template.md` и `docs/dev/tasks/_templates/stage_completion_report_template.md`, затем проверьте, что шаблоны находятся в исходном (долокализационном) виде.
3. Откройте `docs/dev/tasks/docs-cleanup-ru/2026-02-23_docs-cleanup-ru_inventory.md`, затем проверьте раздел `Этап 3: Статус устранения расхождений и русификации`.
