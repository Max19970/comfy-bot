## Контекст задачи

### Запрос и ожидаемый результат [M]

- Кратко о запросе: нужно найти и исправить дефект, при котором включение Hi-Res Fix принудительно включает обычный дополнительный sampler-проход, а при последующем отключении sampler-прохода Hi-Res Fix фактически перестает применяться.
- Ожидаемый результат: Hi-Res Fix работает независимо от флага простого sampler-pass; переключатели в UI не ломают логику применения улучшений.

### Границы работ [M]

- В scope:
  - Логика переключателей улучшений в меню предпросмотра/улучшения.
  - Логика выбора пайплайна выполнения улучшения (какой путь уходит в ComfyUI).
  - Тесты на регрессию для комбинаций `enable_hires_fix` и `enable_sampler_pass`.
- Вне scope [O]:
  - Редизайн меню или изменение текстов UX вне контекста бага.
  - Изменение алгоритмов самого Hi-Res в ComfyUI workflow.

### Основные цели и ограничения [M]

- Файлы/компоненты:
  - `handlers/prompt_editor_handlers_send.py`
  - `handlers/prompt_editor_enhancement.py`
  - `tests/` (новые или обновленные тесты по теме переключателей/улучшения)
- Зоны системы: `backend`, `бот-обработчики`, `интеграция с ComfyUI`, `тесты`.
- Ограничения и допущения [O]:
  - Должна сохраниться обратная совместимость для сценариев, где включен только sampler-pass.
  - Нельзя ухудшать стабильность цепочки постобработки (compression/shrink).
