## Контекст задачи

### Запрос и ожидаемый результат [M]

- Кратко о запросе: проанализировать текущую структуру репозитория и навести порядок в расположении файлов/папок.
- Желаемый результат: логичная и предсказуемая структура каталогов, в которой файлы разложены по назначению, а проект продолжает запускаться и проходить проверки.

### Границы объема [M]

- В рамках задачи:
  - Провести инвентаризацию текущей структуры и выявить файлы, находящиеся не на своем месте.
  - Согласовать целевую структуру папок и карту перемещений.
  - Переместить/переименовать файлы по согласованной карте.
  - Обновить импорты, пути и конфигурации после перемещений.
  - Проверить работоспособность тестами и статическими проверками.
- Вне рамок [O]:
  - Изменение продуктовой логики бота.
  - Добавление новых пользовательских функций.
  - Масштабный редизайн архитектурных слоев вне задачи упорядочивания файлов.

### Основные цели и ограничения [M]

- Файлы/компоненты:
  - Корневые Python-файлы и служебные артефакты (`bot.py`, `bot_core.py`, `app_context.py`, `config.py`, `comfyui_client.py`, `model_downloader.py`, `smart_prompt.py`, `find_russian.py`, `filter_strings.py`, `test_smart_prompt.py`, текстовые артефакты в корне).
  - Папки `handlers/`, `core/`, `application/`, `domain/`, `infrastructure/`, `presentation/`, `docs/`.
- Системы/области: архитектурная организация кода, импорты, конфигурация инструментов (`pytest`, `mypy`, `ruff`), документация по структуре.
- Ограничения и допущения [O]:
  - Не ломать существующую точку входа `bot.py` без явной причины.
  - Изменения должны быть обратимо-проверяемыми через тесты и линтер.
  - Любые перемещения делать этапно, чтобы упростить диагностику регрессий.

### Результаты Этапа 1 - инвентаризация и карта целевого размещения

#### Сводка инвентаризации

- Верхний уровень содержит `41` запись, включая `10` Python-файлов в корне.
- В `handlers/` находится `46` модулей в одном плоском каталоге.
- Найдены служебные и исследовательские артефакты в корне: `find_russian.py`, `filter_strings.py`, `test_smart_prompt.py`, `ru_strings_ast.txt`, `russian_strings.txt`.

#### Выявленные проблемные точки

- В корне смешаны разные типы файлов: entrypoint, инфраструктурные сервисы, конфигурация и вспомогательные скрипты.
- Служебные скрипты и их выходные артефакты не выделены в профильные директории.
- Роль некоторых корневых модулей не отражена структурно по слоям (`application`/`infrastructure`/`core`), что усложняет навигацию.

#### Согласованная целевая схема каталогов

```text
bot.py                         # entrypoint остается в корне
bot_core.py                    # совместимый facade остается в корне
bootstrap/
  app_context.py               # композиция зависимостей и lifecycle
core/
  config.py                    # конфигурация окружения
application/
  model_downloader.py          # прикладной сервис поиска/скачивания моделей
  smart_prompt_service.py      # прикладной сервис smart prompt
infrastructure/
  comfyui_client.py            # клиент ComfyUI API
tools/
  i18n/
    find_russian.py
    filter_strings.py
  manual/
    test_smart_prompt.py
docs/
  dev/
    artifacts/
      i18n/
        ru_strings_ast.txt
        russian_strings.txt
```

#### Карта перемещений (old -> new)

| Старый путь | Новый путь | Этап | Причина |
| --- | --- | --- | --- |
| `app_context.py` | `bootstrap/app_context.py` | Этап 3 | Выделение composition root из корня |
| `config.py` | `core/config.py` | Этап 3 | Единая точка конфигурации в shared-слое |
| `comfyui_client.py` | `infrastructure/comfyui_client.py` | Этап 3 | Явное размещение инфраструктурного адаптера |
| `model_downloader.py` | `application/model_downloader.py` | Этап 3 | Прикладной сервис в соответствующем слое |
| `smart_prompt.py` | `application/smart_prompt_service.py` | Этап 3 | Уточнение роли сервиса и его слоя |
| `find_russian.py` | `tools/i18n/find_russian.py` | Этап 2 | Вынос утилиты локализации из корня |
| `filter_strings.py` | `tools/i18n/filter_strings.py` | Этап 2 | Вынос утилиты анализа строк из корня |
| `test_smart_prompt.py` | `tools/manual/test_smart_prompt.py` | Этап 2 | Явное отделение ручного скрипта от автотестов |
| `ru_strings_ast.txt` | `docs/dev/artifacts/i18n/ru_strings_ast.txt` | Этап 2 | Перенос промежуточного артефакта в docs/dev |
| `russian_strings.txt` | `docs/dev/artifacts/i18n/russian_strings.txt` | Этап 2 | Перенос промежуточного артефакта в docs/dev |

#### Проверка совместимости с архитектурными границами

- Перемещение `comfyui_client.py` в `infrastructure/` соответствует роли адаптера внешней системы.
- Перемещение `model_downloader.py` и `smart_prompt.py` в `application/` сохраняет use-case ориентацию сервисов.
- Перемещение `config.py` в `core/` сохраняет доступность конфигурации без нарушения boundary-правил.
